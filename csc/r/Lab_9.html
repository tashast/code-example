Отбросим столбцы с большим количеством пропущенных значений (25 и более):

plants.02 <- subset(plants.01, select = -c(2, 3, 7))

В оставшей выборке помимо названий растений остальные величины являются факторами, поэтому будем рассматривать нормализацию методом feature-scaling с максимумом 1:

normalize <- function(x) {
    return ((x - min(na.omit(x))) / (max(na.omit(x)) - min(na.omit(x))))
  }
plants.02.fs <- as.data.frame(lapply(plants.02[, 2:29], normalize))

Чтобы не терять наблюдения, заменим NA в тех столбцах, где оно встречается редко (менее 5 раз) на среднее по всему столбцу:

for(i in 1:ncol(plants.02.fs)){
  plants.02.fs[is.na(plants.02.fs[,i]), i] <- mean(plants.02.fs[,i], na.rm = TRUE)
}

Иерархический кластерный анализ:

Составляем матрицу попарных расстояний, проводим кластерный анализ:

clust.fs <- hclust(dist(plants.02.fs), "ward.D")

По результатам кластерного анализа дендрограмму делили на 5 кластеров:

plot(clust.fs)
rect.hclust(clust.fs, k = 5, border = "blue") 

groups.fs.5 <- cutree(clust.fs, k=5)

Второй вариант – на 3 кластера:

plot(clust.fs)
rect.hclust(clust.fs, k = 3, border = "darkgreen") 

groups.fs.3 <- cutree(clust.fs, k=3)

Рассмотрим интерпретацию результатов обоих вариантов кластеризации при помощи решающих деревьев. Сначала 5 классов. Добавим к ненормализованным данным столбец, в котором будет записан номер кластера:

plants.03 <- plants.02[2:29]
cl <- rep(1, nrow(plants.03))
plants.03$cl <- cl
plants.03$cl[groups.fs.5 == 2] <- 2
plants.03$cl[groups.fs.5 == 3] <- 3
plants.03$cl[groups.fs.5 == 4] <- 4
plants.03$cl[groups.fs.5 == 5] <- 5

Для оценки качества работы решающих деревьев разобьём выборку на тестовое и обучающее множество:

set.seed(3141)

Количество ошибок на тестовом множестве сравнимо с ошибками на обучающем – работа решающего дерева считается удовлетворительной. Интерпретация результата:

К 5-му кластеру относятся растения, у которых переменная endozoo равна 1 – способ распространения семян через пищеварительный тракт животного. В остальных кластерах растения, у которых эта переменная равна 0. В 4-м кластере растения с переменной suman, равной 1 – цветение летом (также endozoo = 0). В остальных кластерах растения без летнего цветения. Далее в 3-м кластере растения со значением переменной semiros, равной 1 – полурозоцветные (также endozoo = 0, suman = 0). Во втором кластере не очень высокие растения с endozoo, suman, semiros, равными 0. В первом кластере самые высокие растения – деревья (endozoo, suman, semiros равны 0).

Тоже не очень много ошибок. Интерпретация результата:

Растения из первого класса имеют значение переменной lign, равное 1 – растения, которые являются деревьями. Остальные классы не являются деревьями. Растения второго класса не имеют летнего цветения, растения третьего класса с летним цветением.

Если не разбивать на тестовое и обучающее множество для трёх кластеров, дерево получается сложнее и его нужно обрезать. Построим дерево для трёх кластеров на всей выборке:

res_3 <- rpart(cl ~., data = plants.04, method="class", control = rpart.control(minsplit=10, minbucket=5, maxdepth=6))
rpart.plot(res_3, type = 2, extra = 1)

Листьев сильно больше, чем классов – неудобно интерпретировать. Увеличим минимальное число элементов в листе:

res_4 <- rpart(cl ~., data = plants.04, method="class", control = rpart.control(minsplit=15, minbucket=5, maxdepth=6))
rpart.plot(res_4, type = 2, extra = 1)

Получилось дерево небольшой глубины и не очень много ошибок, всего 4 листа. Разбиение по endozoo является незначительным, уберём его, уменьшив глубину дерева:
