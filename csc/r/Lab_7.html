Пропущенных данных нет. Программа-минимум состоит в классификации оливковых масел для трёх регионов (переменная region) на основании жирных кислот (столбцы с 4 по 11).

Разобьём выборку на обучающее и тестовое множество в пропорции 2:1, убрав столбцы, отвечающие за область:

set.seed(3141)
test_num <- sample(1:nrow(olive), 191, replace = FALSE)
olive_train <- olive[-test_num, 4:11]
olive_test <- olive[test_num, 4:11]

Код класса – переменная region:

cl <- olive[-test_num, 2]

Попробуем работу метода k-ближайших соседей для k = 3:

library(class)
res_01 <- knn(olive_train, olive_test, cl, k = 3)

Проверяем результат работы на тестовом множестве:

Суммирование производится по столбцам. Всего 4 ошибки. Проверим число ошибок для разного количества соседей:

errors <- rep(0, 15)

for (i in 1:15){
    tmp <- knn(olive_train, olive_test, cl, k = i)
    errors[i] <- sum(tmp != olive[test_num, 2])
}
errors
##  [1]  2  6  4  5  4  3  3  3  3  3  5  7  6  8 10

Для 1 или 3 соседей результат неплохой, для 6-10 одинаковое число ошибок, а чем больше значения после – тем хуже. Хорошее число соседей 1, 3, либо 6. Лучше не брать большое число соседей (6-10), т.к. при другом разбиении на обучающее и тестовое множество большое число соседей может больше ошибок.
